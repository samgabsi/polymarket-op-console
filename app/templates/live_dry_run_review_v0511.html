State meanings

  • validated_ready: latest receipt matches the packet hash and has no warnings.
  • validated_with_warnings: latest receipt matches the packet hash, but packet or receipt warnings need human review.
  • needs_dry_run_receipt: packet exists and is packageable, but no receipt has been saved.
  • stale_dry_run_receipt: latest receipt no longer matches the current packet hash, status, intent, or authorization snapshot.
  • dry_run_blocked: latest receipt is blocked or reports unsafe dry-run flags.
  • packet_blocked: saved packet is not currently suitable for dry-run review.

What this does not do

This review board is a derived local report. It does not save approvals, does not authorize trading, does not sign payloads, does not submit or cancel orders, does not call Polymarket, and does not automate execution.

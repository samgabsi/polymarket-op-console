Trade Ticket Preview

POST JSON to /api/v2/live/ticket/preview to create a draft ticket, calculate notional/max loss, and run strict risk checks. POST the same payload to /api/v2/live/order/submit only after human approval, warning acknowledgement, and typed confirmation.

FieldRequiredNotes
market_idYesGamma market or condition identifier.
token_idYesCLOB token/asset id for selected outcome.
sideYesBUY or SELL.
priceYesStrictly between 0 and 1.
sizeYesOutcome shares.
order_typeNoGTC default; FOK/FAK require market-order permission.
human_approvalSubmit onlyMust be true when approval is required.
acknowledge_warningsSubmit onlyMust be true when warnings exist.
confirmation_phraseSubmit onlyMust exactly match configured phrase; default is LIVE ORDER APPROVED.

Emergency Controls

POST /api/v2/live/emergency with action force_read_only, disable_new_live_orders, record_kill_switch, or cancel_all_preview. Runtime environment changes remain explicit and auditable through Settings/.env so hidden background mutation cannot arm or disarm trading unexpectedly.

Force read-only

Records an emergency read-only action and directs the operator to persist READ_ONLY=true and submit gates false.

Disable new live orders

Records a control action for turning off submit gates.

Cancel all preview

Preview-only safety action; targeted cancellation remains confirmed and audited.

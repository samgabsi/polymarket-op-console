# Visual QA Checklist — v3.3.0-real

Use this checklist before publishing screenshots, recording a demo, or tagging a release.

## Command center

- Safety cards are readable.
- Live Armed, Read-only, and Kill Switch badges are clear.
- Data Health, Research/Strategy, Monitoring, Portfolio, and Governance sections scan correctly.
- Next Actions are understandable.
- Missing-prerequisite findings show severity, title, explanation, and action.

## Search

- Search page explains local-only behavior.
- Result rows show type, title, snippet, status, tags, and quick link.
- Empty states are useful.
- Filter metadata is available.

## Graph

- Graph summary cards show nodes/edges/filter counts.
- Relationship table is readable.
- Empty state is understandable.
- Exports work.

## Workflows and briefs

- Workflow template sections are visible.
- Pre-trade packet output is readable.
- Market brief includes support, contradiction, missing research, and next actions.
- Thesis health includes evidence/counter-evidence/stale state.
- Portfolio brief includes concentration and unknown-data sections.
- Operator review includes next actions.

## Responsive layout

- Narrow-width view remains readable.
- Tables do not obscure safety warnings.
- Dark-mode contrast is acceptable.

## Safety and secrets

- No secrets are visible.
- Demo data is clearly fake.
- No page implies financial advice.
- No page implies v3 workflows can place/cancel/approve orders.
- Export buttons are clear and safe.

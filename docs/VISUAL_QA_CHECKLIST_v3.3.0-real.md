# Visual QA Checklist — v3.3.0-real

Use this checklist before publishing screenshots or presenting the demo.

## Command center

- [ ] System Safety cards are visible above the fold.
- [ ] Operator Attention Queue is easy to scan.
- [ ] Workbench Shortcuts are obvious.
- [ ] Intelligence Summary is readable.
- [ ] Safe Next Actions do not look like trade instructions.

## Navigation

- [ ] Sidebar groups are visible: Operate, Analyze, Build Thesis, Govern, Output.
- [ ] v2 compatibility links work.
- [ ] Active navigation state is clear.

## Search and graph

- [ ] Search result filters are readable.
- [ ] Empty search state is useful.
- [ ] Graph filters are visible.
- [ ] Relationship table is readable.

## Workflows and reports

- [ ] Workflow cards explain purpose and read-only status.
- [ ] Packets/briefs show blockers, warnings, unknowns, and safety statements.
- [ ] Export buttons are visible but not misleading.

## Analytics

- [ ] Analytics cards are readable.
- [ ] Calibration sample size and unknown counts are visible.
- [ ] Learning report language is descriptive, not predictive.

## Forms and tables

- [ ] Tables have clear headings and captions.
- [ ] Filters/search fields are labeled.
- [ ] Row actions are understandable.
- [ ] Dangerous actions are visually distinct.

## Safety and secrets

- [ ] No secrets are visible.
- [ ] Demo mode is clearly fake.
- [ ] Read-only/live-armed/kill-switch states are visible.
- [ ] No misleading financial-advice language appears.

## Responsive/accessibility

- [ ] Narrow-width layout remains usable.
- [ ] Keyboard focus is visible.
- [ ] Text contrast is readable.
- [ ] Color is not the only status indicator.
